## 账户管理功能

### 功能概述

Antigravity Agent 的核心功能是管理多个 Antigravity 账户。应用程序会自动检测并备份当前登录的 Antigravity 账户信息，让您可以轻松在多个账户之间切换。

### 自动账户检测

当您在 Antigravity 中登录账户后，Antigravity Agent 会：

1. **自动检测**：实时监控 Antigravity 数据库变化
2. **自动备份**：将账户信息（包括 API 密钥、用户设置等）保存到本地
3. **自动更新**：如果账户信息发生变化，自动更新备份

### 账户信息显示

在主界面的"用户管理"区域，您可以看到所有已备份的账户：

- **头像**：显示账户的个人头像
- **用户名**：显示账户的用户名（部分脱敏显示）
- **邮箱**：显示账户邮箱（部分脱敏显示）
- **创建时间**：显示账户首次备份的时间
- **当前标识**：当前正在使用的账户会显示"当前使用"标签和蓝色高亮

### 查看账户详情

点击任意账户卡片，可以查看该账户的详细信息：

- **基本信息**：用户名、邮箱、头像
- **API 密钥**：完整的 API Key（可复制）
- **时间信息**：创建时间、最后切换时间
- **用户设置**：完整的用户配置数据（JSON 格式）

所有敏感信息都支持一键复制功能，方便您在需要时使用。

### 删除账户备份

如果您不再需要某个账户的备份，可以：

1. 找到要删除的账户卡片
2. 点击右侧的垃圾桶图标
3. 在确认对话框中点击"确认删除"

**注意**：删除操作不可撤销，但不会影响 Antigravity 中的实际账户。

### 清空所有备份

如果需要清空所有账户备份：

1. 点击"用户管理"标题右侧的垃圾桶图标
2. 在确认对话框中确认操作
3. 所有账户备份将被永久删除

**警告**：此操作会删除所有账户备份，请谨慎使用。


## 账户切换操作

### 切换到其他账户

Antigravity Agent 让您可以快速在不同账户之间切换，无需手动输入账户信息。

#### 切换步骤

1. **选择账户**：在用户列表中找到要切换的账户
2. **点击切换按钮**：点击账户卡片右侧的"切换"按钮
3. **等待完成**：应用会显示"正在切换到用户..."的提示
4. **自动应用**：切换完成后，Antigravity 会自动使用新账户

#### 切换过程说明

切换账户时，应用程序会：

1. 读取目标账户的备份数据
2. 将账户信息写入 Antigravity 数据库
3. 更新 API 密钥和用户设置
4. 刷新界面显示当前账户

**注意**：
- 切换账户不需要重启 Antigravity
- 切换后，Antigravity 会立即使用新账户的配置
- 原账户的信息已经备份，可以随时切换回来

#### 切换后验证

切换完成后，您可以：

- 查看界面上的"当前使用"标签，确认已切换到正确账户
- 在 Antigravity 中检查当前登录的账户信息
- 查看配额仪表盘，确认显示的是新账户的配额

### 配额仪表盘

当有账户处于"当前使用"状态时，界面顶部会显示该账户的配额信息：

- **模型列表**：显示所有可用的 AI 模型
- **使用情况**：每个模型的已用量和总配额
- **进度条**：直观显示配额使用百分比
- **颜色标识**：
  - 绿色：使用量正常（< 80%）
  - 黄色：使用量较高（80% - 95%）
  - 红色：配额即将用尽（> 95%）


## 登录新账户流程

### 功能说明

"登录新账户"功能允许您在 Antigravity 中登录一个全新的账户，同时自动备份当前账户信息。

### 使用场景

- 需要添加一个新的 Antigravity 账户
- 想要使用不同的账户进行工作
- 需要测试多个账户的功能

### 操作步骤

#### 1. 点击"登录新账户"按钮

在应用顶部工具栏，点击"登录新账户"按钮。

#### 2. 确认操作

系统会显示确认对话框，说明此操作将：

- 关闭所有 Antigravity 进程
- 自动备份当前账户信息
- 清除 Antigravity 用户信息
- 自动重新启动 Antigravity

点击"确认"继续，或点击"取消"放弃操作。

#### 3. 等待处理

应用会显示"正在备份当前用户并注销..."的提示，此过程包括：

1. **备份当前账户**：保存当前账户的所有信息
2. **关闭进程**：安全关闭 Antigravity 进程
3. **清除数据**：清除当前的登录信息
4. **重启服务**：自动重新启动 Antigravity

#### 4. 在 Antigravity 中登录

当 Antigravity 重新启动后：

1. 打开 Antigravity 应用
2. 使用新账户的凭据登录
3. 完成登录流程

#### 5. 自动保存新账户

登录完成后，Antigravity Agent 会：

- 自动检测到新账户
- 自动备份新账户信息
- 在用户列表中显示新账户

### 注意事项

- **自动备份**：原账户会自动备份，不会丢失
- **进程管理**：应用会自动处理 Antigravity 的启动和关闭
- **等待时间**：整个过程可能需要几秒到十几秒，请耐心等待
- **手动刷新**：如果新账户没有自动显示，可以尝试重启应用

### 常见问题

**Q: 原账户的数据会丢失吗？**
A: 不会。原账户会自动备份，您可以随时切换回来。

**Q: 如果 Antigravity 没有自动启动怎么办？**
A: 可以手动启动 Antigravity，或在应用中点击"重试"按钮。

**Q: 可以同时登录多个账户吗？**
A: 不可以。Antigravity 一次只能登录一个账户，但您可以快速切换。


## 导入导出配置

### 功能概述

导入导出功能允许您：

- **备份**：将所有账户信息导出为加密文件
- **迁移**：在不同设备间转移账户配置
- **恢复**：从备份文件恢复账户信息

所有导出的配置文件都经过加密，确保您的账户信息安全。

### 导出配置

#### 导出步骤

1. **点击导出按钮**：在顶部工具栏点击"导出"按钮
2. **等待数据收集**：应用会收集所有账户的完整信息
3. **设置密码**：在密码对话框中输入加密密码
   - 密码要求：至少 8 个字符
   - 建议使用强密码，包含字母、数字和特殊字符
4. **选择保存位置**：选择配置文件的保存路径
5. **完成导出**：文件保存成功后会显示确认消息

#### 导出文件说明

- **文件名格式**：`antigravity_encrypted_config_YYYY-MM-DD-HH-mm-ss.enc`
- **文件扩展名**：`.enc`（表示加密文件）
- **文件内容**：包含所有账户的完整信息（加密存储）

#### 导出的数据内容

导出文件包含：

- 所有账户的邮箱和用户名
- API 密钥
- 用户设置和配置
- 账户创建时间和最后切换时间
- 用户头像 URL

**注意**：
- 如果没有任何账户，导出按钮会被禁用
- 导出过程中请勿关闭应用
- 请妥善保管导出文件和密码

### 导入配置

#### 导入步骤

1. **点击导入按钮**：在顶部工具栏点击"导入"按钮
2. **选择配置文件**：选择之前导出的 `.enc` 文件
3. **输入密码**：输入导出时设置的密码
4. **等待解密和恢复**：应用会自动解密并恢复账户信息
5. **查看结果**：导入完成后，所有账户会显示在用户列表中

#### 导入行为说明

- **合并模式**：导入的账户会与现有账户合并
- **覆盖规则**：如果邮箱相同，会更新为导入文件中的信息
- **自动刷新**：导入完成后自动刷新用户列表

#### 错误处理

如果导入失败，可能的原因：

- **密码错误**：请确认输入的密码正确
- **文件损坏**：配置文件可能已损坏，请使用其他备份
- **格式不兼容**：文件可能不是有效的配置文件

### 安全建议

1. **定期备份**：建议定期导出配置文件作为备份
2. **密码管理**：使用密码管理器保存导出密码
3. **安全存储**：将配置文件存储在安全的位置
4. **多重备份**：在多个位置保存备份文件
5. **测试恢复**：定期测试导入功能，确保备份可用


## 系统托盘使用

### 功能说明

系统托盘功能允许应用在后台运行，关闭主窗口时不会退出程序，而是最小化到系统托盘。

### 启用系统托盘

1. 点击右上角的设置图标（齿轮图标）
2. 在设置对话框中找到"系统托盘"选项
3. 点击开关启用系统托盘功能

### 系统托盘行为

#### 启用后的行为

- **关闭窗口**：点击窗口关闭按钮时，应用会最小化到托盘而不是退出
- **托盘图标**：在系统托盘区域显示应用图标
- **后台运行**：应用继续在后台运行，监控账户变化

#### 禁用后的行为

- **关闭窗口**：点击窗口关闭按钮时，应用会完全退出
- **无托盘图标**：系统托盘不显示应用图标

### 托盘菜单功能

右键点击托盘图标，可以看到以下选项：

- **显示窗口**：恢复主窗口显示
- **退出应用**：完全退出应用程序

### 恢复主窗口

从托盘恢复主窗口的方法：

1. **单击托盘图标**：直接单击托盘图标
2. **右键菜单**：右键点击托盘图标，选择"显示窗口"

### 使用场景

系统托盘功能适合以下场景：

- **后台监控**：需要应用持续监控账户变化
- **快速访问**：需要随时快速打开应用
- **节省任务栏空间**：不想在任务栏显示应用窗口
- **多任务工作**：在多个应用间频繁切换

### 注意事项

- 系统托盘设置会自动保存，下次启动时生效
- 如果找不到托盘图标，请检查系统托盘设置（可能被隐藏）
- 某些操作系统可能需要手动配置托盘图标显示


## 应用设置

### 打开设置

点击应用右上角的设置图标（齿轮图标）打开设置对话框。

### 可执行文件路径

#### 功能说明

指定 Antigravity 可执行文件的位置。应用会使用此路径来启动和管理 Antigravity 进程。

#### 自动检测

应用会自动检测 Antigravity 的安装位置：

- **Windows**：检查常见安装路径和注册表
- **macOS**：检查 Applications 文件夹和常见位置

如果自动检测成功，会显示"自动检测"标识。

#### 手动设置

如果自动检测失败，或需要使用特定版本：

1. 点击可执行文件路径右侧的文件图标
2. 浏览并选择 Antigravity 可执行文件
   - Windows: `Antigravity.exe`
   - macOS: `Antigravity.app`
3. 应用会验证文件有效性
4. 设置成功后会显示确认消息

### 系统托盘设置

#### 启用/禁用系统托盘

- **开关位置**：设置对话框中的"系统托盘"选项
- **功能说明**：关闭窗口时最小化到托盘
- **默认状态**：默认启用

详细说明请参考 [系统托盘使用](#系统托盘使用) 章节。

### 静默启动设置

#### 功能说明

启用静默启动后，应用启动时会自动隐藏主窗口，直接最小化到系统托盘。

#### 使用场景

- **开机自启动**：配合系统开机自启动功能使用
- **后台运行**：不需要立即使用应用界面
- **减少干扰**：避免启动时弹出窗口

#### 启用方法

1. 打开设置对话框
2. 找到"静默启动"选项
3. 点击开关启用功能

#### 注意事项

- 静默启动需要配合系统托盘功能使用
- 如果系统托盘被禁用，静默启动会自动禁用
- 启用后，下次启动应用时会直接进入托盘

### 版本信息

设置对话框标题栏会显示当前应用版本号，格式为 `v1.0.3`。

### 设置保存

所有设置会自动保存到配置文件，下次启动时自动加载：

- **Windows**: `%APPDATA%\.antigravity-agent\config.json`
- **macOS**: `~/.config/.antigravity-agent/config.json`
- **Linux**: `~/.config/.antigravity-agent/config.json`


## 日志查看和问题反馈

### 日志系统

Antigravity Agent 使用双层日志系统记录应用运行信息：

1. **控制台日志**：开发模式下在控制台输出
2. **文件日志**：持久化保存到日志文件

### 日志文件位置

日志文件保存在以下位置：

- **Windows**: `%APPDATA%\.antigravity-agent\logs\`
- **macOS**: `~/.config/.antigravity-agent/logs/`
- **Linux**: `~/.config/.antigravity-agent/logs/`

### 日志文件命名

日志文件按日期命名，格式为：

```
antigravity-agent-YYYY-MM-DD.log
```

例如：`antigravity-agent-2025-12-04.log`

### 日志内容

日志记录包括：

- **时间戳**：精确到毫秒的时间信息
- **日志级别**：INFO、WARN、ERROR、DEBUG
- **模块名称**：标识日志来源的模块
- **操作类型**：具体的操作或事件
- **详细信息**：相关的数据和错误信息

### 日志级别说明

- **INFO**：正常操作信息（如账户切换、备份成功）
- **WARN**：警告信息（如检测失败、数据异常）
- **ERROR**：错误信息（如操作失败、异常情况）
- **DEBUG**：调试信息（开发模式下的详细信息）

### 查看日志

#### 方法 1：直接打开日志文件

1. 导航到日志文件目录
2. 使用文本编辑器打开最新的日志文件
3. 查看相关的日志记录

#### 方法 2：使用开发者工具

1. 按 `Ctrl+Shift+I`（Windows/Linux）或 `Cmd+Option+I`（macOS）
2. 切换到 Console 标签
3. 查看实时日志输出

## 常见问题

### 账户管理相关

**Q: 为什么我的账户没有自动显示？**
A: 请确保：
- Antigravity 正在运行
- 已在 Antigravity 中成功登录账户
- 应用有权限访问 Antigravity 数据库
- 尝试重启 Antigravity Agent

**Q: 删除账户备份会影响 Antigravity 中的账户吗？**
A: 不会。删除备份只是删除本地保存的账户信息，不会影响 Antigravity 中的实际账户。

**Q: 可以管理多少个账户？**
A: 理论上没有限制，但建议不超过 20 个账户以保持良好的性能。

### 账户切换相关

**Q: 切换账户需要重启 Antigravity 吗？**
A: 不需要。切换操作会直接更新 Antigravity 的数据库，无需重启。

**Q: 切换账户后原账户的数据会丢失吗？**
A: 不会。原账户的信息已经备份，可以随时切换回来。

**Q: 切换账户失败怎么办？**
A: 请检查：
- Antigravity 是否正在运行
- 目标账户的备份数据是否完整
- 查看日志文件了解具体错误
- 尝试重启应用后再次切换

### 导入导出相关

**Q: 忘记导出密码怎么办？**
A: 很遗憾，加密文件无法在不知道密码的情况下解密。建议：
- 使用密码管理器保存密码
- 在安全的地方记录密码
- 定期创建新的备份

**Q: 可以在不同设备间共享配置文件吗？**
A: 可以。导出的配置文件是跨平台的，可以在 Windows、macOS 和 Linux 之间共享。

**Q: 导入配置会覆盖现有账户吗？**
A: 如果邮箱相同，会更新为导入文件中的信息。如果邮箱不同，会添加为新账户。

### 系统托盘相关

**Q: 找不到托盘图标怎么办？**
A: 请检查：
- 系统托盘功能是否已启用
- 操作系统的托盘设置（某些系统会隐藏图标）
- 尝试重启应用

**Q: 关闭窗口后如何重新打开？**
A: 单击系统托盘图标，或右键选择"显示窗口"。

**Q: 如何完全退出应用？**
A: 右键点击托盘图标，选择"退出应用"。

### 进程管理相关

**Q: "登录新账户"功能做了什么？**
A: 该功能会：
1. 备份当前账户
2. 关闭 Antigravity 进程
3. 清除登录信息
4. 重新启动 Antigravity

**Q: Antigravity 没有自动启动怎么办？**
A: 可以手动启动 Antigravity，或在应用中点击"重试"按钮。

**Q: 应用显示"Antigravity 未在运行"怎么办？**
A: 请：
- 手动启动 Antigravity
- 点击"手动刷新"按钮
- 检查 Antigravity 是否正确安装
- 在设置中检查可执行文件路径

### 配置和设置相关

**Q: 设置会自动保存吗？**
A: 是的。所有设置会自动保存到配置文件，下次启动时自动加载。

**Q: 如何重置所有设置？**
A: 删除配置文件目录中的 `config.json` 文件，应用会使用默认设置。

**Q: 静默启动不生效怎么办？**
A: 请确保：
- 系统托盘功能已启用
- 静默启动开关已打开
- 重启应用后生效

### 性能和稳定性

**Q: 应用占用内存过高怎么办？**
A: 这是正常现象。应用使用 Tauri 框架，内存占用通常在 50-150MB 之间。如果超过 200MB，可以尝试重启应用。

**Q: 应用启动很慢怎么办？**
A: 可能的原因：
- 首次启动需要检测 Antigravity 路径
- 账户数量过多
- 系统资源不足
- 尝试清理日志文件

**Q: 应用崩溃怎么办？**
A: 请：
1. 查看日志文件了解崩溃原因
2. 尝试重启应用
3. 如果持续崩溃，提交问题反馈

### 数据安全

**Q: 账户信息存储在哪里？**
A: 账户备份存储在：
- Windows: `%APPDATA%\.antigravity-agent\backups\`
- macOS: `~/.config/.antigravity-agent/backups/`
- Linux: `~/.config/.antigravity-agent/backups/`

**Q: 账户信息是否加密？**
A: 本地备份文件不加密，但导出的配置文件使用密码加密。建议：
- 使用操作系统的磁盘加密功能
- 定期导出加密备份
- 妥善保管导出文件

**Q: 如何确保数据安全？**
A: 建议：
- 定期导出加密备份
- 使用强密码
- 不要在公共设备上使用
- 及时更新应用版本
